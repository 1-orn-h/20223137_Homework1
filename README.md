## 과제 목표

다음 리눅스 명령어에 대해서 조사해 작성할 예정입니다
|  Name  |      Fuction         |
|  :-:   |         ---          |
|  top |시스템의 실시간 성능 확인 가능|
|  ps  |현재 실행중인 프로세스 상태 확인|
|  jobs  |쉘에서 실행된 백그라운드 작업 목록을 보여주는 기능|
|kill |  프로세스에 시그널(signal)을 보내 프로세스를 제어하는 기능|

## 1. top 명령어
+ 용도 : 실시간으로 시스템의 전반적인 상태(CPU, 메모리)와 프로세스들의 자원 사용량 확인
+ 특징 : 실행하면 화면이 3초마다 계속 갱신되는 동적인 명령어
+ 자주 사용하는 키
    + q : 종료
    + P : CPU 사용률 순으로 정렬(기본값)
    + M : 메모리 사용률 순으로 정렬
     ![top 실행 화면](https://github.com/1-orn-h/20223137_Homework1/blob/main/top.png)

## 2. ps 명령어
+ 용도 : 현재 시스템에 실행 중인 프로세스들의 스냅샷을 볼 때 사용
+ 특징 : top과 달리 한 번 출력하고 나면 멈추는 정적인 목록임. 주로 다른 명령어와 함께 파이프라인을 써서 특정 프로세스를 찾을 때 많이 씀.
+ 주요 옵션 :
    + -e 또는 -A : 모든 프로세스 표시
    + f : 트리구조 형태로 표시
    + q : 터미널과 관련된 모든 프로세스 표시
    + u : 사용자 기반 포맷으로 자세한 정보를 표시
       ![ps 실행 화면](https://github.com/1-orn-h/20223137_Homework1/blob/main/ps.png)

## 3. jobs 명령어
+ 용도 : 현재 로그인된 셸에서 백그라운드나 중지된 상태로 실행 중인 명령어들의 목록을 확인할 때 씀
+ 특징 : 직접 실행한 명령어에 대해서만 보여줌. 다른 사람이 실행한 프로세스는 ps로 봐야함
+ 상태 표시:
    + Running : 백그라운드에서 실행 중
    + Stopped : 실행이 일시 중지 됨
    + Done : 작업이 완료 됨

 ## 4. kill 명령어
 + 용도 : 특정 프로세스에게 시그널을 보내서 종료하거나 제어할 때 사용
 + kill 명령어로 종료가 안 될시, 강제 종료 시그널인
   `kill -9 [PID]
    입력
